Development Guidelines

  • Code Style: Follow industry-standard coding practices and naming conventions.
  • Version Control: Always use Git for version control. Push changes to feature branches and submit pull requests for code reviews.
  • Documentation: Document your code well. Use JSDoc or equivalent standards to ensure clarity.
  • Testing: Write tests for new code. Run tests locally before pushing changes.
